Support center

Get help the right way.

The Veyloria Support Center is designed to keep help requests organized, safe and useful. Before opening a ticket, users should check the documentation, command directory and known information pages. If the issue still needs staff review, the official Discord server is the correct place to ask for help.

Good support starts with clear information. The more specific the report is, the easier it is for staff to understand what happened, reproduce the issue and provide a useful answer without unnecessary back-and-forth.

Use only official support paths.

Veyloria staff will never ask for Discord tokens, bot tokens, passwords, recovery codes, payment credentials, private keys or hosting credentials. Do not trust private messages that claim they can fix your issue faster by asking for sensitive information.

Support process

How a good support request works.

A support request should explain the problem clearly enough for someone else to understand it without guessing. If the issue is related to a command, include the command name, what you expected, what happened and where it happened.

1

Check public information

Read docs, command details, support policy and status information before opening a ticket.

2

Prepare the details

Gather the command, channel, expected result, actual result and safe screenshots or error text.

3

Open support

Use the official Discord support server and keep the conversation inside the correct support channel or ticket.

Bug report template

Copy this when reporting a bug.

Use this format when a command does not work as expected or a feature behaves incorrectly.

Command: What I expected: What actually happened: Channel: Server ID if requested by staff: Screenshot or error text: Does it happen every time: Did anything change before the issue:

Setup help template

Copy this for setup issues.

Use this format when Veyloria is not responding, roles are not being assigned or permissions appear incorrect.

Problem: What page or command I followed: Bot role position checked: yes/no Required permissions checked: yes/no Affected channel: Affected role: What I already tried: Screenshot if safe:

Before opening a ticket

Quick checklist.

Most basic issues can be solved faster if these points are checked first.

Check the command directory. Make sure the command exists and you understand the required access level.
Check the docs page. Permission and role hierarchy issues are usually explained there.
Check the status page. If many commands are unavailable, it may be maintenance or a wider issue.
Use the correct server channel. If economy commands are restricted to a specific channel, they may not work elsewhere.
Do not open duplicate tickets. One clear ticket is better than multiple short messages in different places.
Never share secrets. Staff does not need your tokens, passwords or payment credentials to help with normal support.

Can I DM staff for support?

Use the official support server instead. Public support paths are safer, easier to track and reduce impersonation risks.

How fast will support answer?

Response time depends on staff availability and issue complexity. Clear reports are usually easier to review than vague messages.

Can I report premium payment questions?

Yes, but payments are not active until the official bot release. Use the support server for premium launch questions and updates.

What if someone sends me a fake Veyloria link?

Do not click it. Take a screenshot, copy the user ID if possible and report it through the official support server.
